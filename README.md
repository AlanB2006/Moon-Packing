# Moon Packing

[![Tests](https://github.com/AlanB2006/Moon-Packing/actions/workflows/tests.yml/badge.svg)](https://github.com/AlanB2006/Moon-Packing/actions/workflows/tests.yml)
![Zenodo DOI forthcoming](https://img.shields.io/badge/Zenodo-DOI%20forthcoming-blue)

Reproducibility repository for **“Tidal evolution of packed moon systems around an Earth-mass planet”** by Alan Briseno, Billy Quarles, and Marialis Rosario-Franco, forthcoming in *The Astrophysical Journal*.

## Scientific purpose

This study asks how many equal-mass moons an Earth-mass planet at 1 au from a Sun-mass star can retain when both mutual gravity and tidal evolution are included. The calculations use [REBOUND](https://rebound.readthedocs.io/) with the IAS15 integrator and the `tides_spin` model in [REBOUNDx](https://reboundx.readthedocs.io/). The paper considers:

| Architecture | Number of moons | Tidal time lags |
|---|---:|---:|
| Luna-mass | 2 | 0, 100, 698 s |
| Pluto-mass | 3 | 0, 100, 698 s |
| Ceres-mass | 5 | 0, 100, 698 s |

The supplied tables reproduce the lifetime/eccentricity sweeps and eccentricity-variation maps. New integrations can be launched through command-line scripts, including small examples suitable for a laptop and array-job examples for a cluster.

## Installation

Python 3.10–3.12 is recommended. The exact software versions reported in the manuscript are REBOUND 4.4.6 and REBOUNDx 4.4.1.

### Conda

```bash
conda env create -f environment.yml
conda activate moon-packing
python -m pip install -e .
```

### pip

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[simulation,test]'
```

## Reproduce figures from the archived tables

The following command does not rerun the expensive N-body grids. It reads the tables in `data/` and writes publication-style PDFs to `figures/reproduced/`:

```bash
moon-packing-figures --figures 2 3 4 8 9 10
```

Equivalent command without installing entry points:

```bash
PYTHONPATH=src python scripts/reproduce_figures.py --figures 2 3 4 8 9 10
```

Individual examples:

```bash
# Luna-mass lifetime and maximum-eccentricity panels (paper Figure 2)
moon-packing-figures --figures 2

# Luna, Pluto, and Ceres eccentricity-variation maps (paper Figures 8–10)
moon-packing-figures --figures 8 9 10
```

See [`docs/FIGURE_MAP.md`](docs/FIGURE_MAP.md) for the required input file and command for every figure.


### `Fig02.py` simulation archives

The `Fig02.py` workflow also depends on REBOUND SimulationArchive binary files (`*.bin`). Those files are distinct from the comma-separated sweep and eccentricity-map tables described above. The archive files are required when running the `Fig02.py` workflow directly, but they are not required for the table-based `moon-packing-figures --figures 2 3 4 8 9 10` command.

See [`data/Simulation_Archives/README.md`](data/Simulation_Archives/README.md) and [`docs/REPRODUCIBILITY.md`](docs/REPRODUCIBILITY.md) for details.

## Run a small simulation example

A short two-moon sweep finishes far sooner than the full production grid:

```bash
moon-packing-run-grid \
  --moon-type Luna \
  --n-moons 2 \
  --tau 100 \
  --beta-start 6.0 \
  --beta-stop 6.2 \
  --beta-step 0.1 \
  --max-orbits 1000 \
  --workers 2 \
  --output data/generated/luna_demo.csv
```

A single time series for a manuscript-style panel:

```bash
moon-packing-run-timeseries \
  --moon-type Luna \
  --n-moons 2 \
  --tau 698 \
  --beta 8.2 \
  --years 25 \
  --output data/generated/timeseries/luna_beta8.2_tau698.npz
```

Validate the REBOUNDx constant-time-lag implementation against the secular model used in the paper:

```bash
moon-packing-validate-tides --years 30000
moon-packing-figures --figures 1
```

## Reproduce the production grids

The published sweep integrations run to \(10^7\) inner-moon periods, and the eccentricity maps run to \(10^5\) periods. These are computationally expensive and are intended for a cluster.

```bash
# Full Luna sweep for Earth-like dissipation
moon-packing-run-grid \
  --moon-type Luna --n-moons 2 --tau 698 \
  --beta-start 3.5 --beta-stop 9.0 --beta-step 0.01 \
  --max-orbits 1e7 --workers 32 \
  --output data/generated/luna_tau698_full.csv

# Example eccentricity-map tile
moon-packing-run-grid \
  --moon-type Luna --n-moons 2 --tau 698 \
  --beta-start 3.5 --beta-stop 9.0 --beta-step 0.01 \
  --e-start 0.0 --e-stop 0.30 --e-step 0.01 \
  --max-orbits 1e5 --workers 32 \
  --output data/generated/luna_map_tau698.csv
```

For a scheduler-friendly pattern, see [`examples/slurm_array.sh`](examples/slurm_array.sh). Production values and the mapping to manuscript methods are documented in [`docs/REPRODUCIBILITY.md`](docs/REPRODUCIBILITY.md).

## Important reproducibility note

The archived simulation script stops when a moon’s **semimajor axis** exceeds `0.4 R_H`; the revised manuscript describes the criterion in terms of **apocenter**. The command-line code defaults to `--outer-boundary semimajor-axis` so that reruns match the archived tables. Use `--outer-boundary apocenter` to implement the wording in the revised manuscript. This choice should be resolved before the final Zenodo release.

The package also retains the exact semimajor-axis placement expression used by the archived scripts (`--spacing-mode archived`). A recursive implementation of the equation as written in the manuscript is available with `--spacing-mode recursive` for sensitivity checks.

## Tests

```bash
python -m pytest
```

The GitHub Actions workflow runs unit tests and regenerates a small subset of figures without requiring REBOUND. Simulation tests are skipped when REBOUND/REBOUNDx are unavailable.

## Data integrity

`data/MANIFEST.csv` lists every archived data file, byte size, row count, and SHA-256 checksum. Verify the full repository payload with:

```bash
sha256sum --check checksums.sha256
```
